# HomePort API Compatibility Gateway Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let existing cloud SDKs keep working by routing familiar cloud APIs to sovereign/open-source targets.

**Architecture:** Prefer existing compatible backends first: MinIO for S3, Scylla Alternator for DynamoDB, Redis protocol for Redis. Add thin compatibility proxies only where no installed backend already speaks the expected API.

**Tech Stack:** Go HTTP reverse proxy, SigV4 verification where needed, existing storage/database/messaging targets, conformance tests using official SDKs.

## Task 1: Add adapter registry

- [ ] Define `Adapter` with `Provider()`, `Service()`, `Routes()`, `TargetEnv()`, `ConformanceChecks()`.
- [ ] Register built-ins: `s3`, `dynamodb`, `redis`, `sqs`, `sns`, `secretsmanager`, `ssm`, `cloudwatchlogs`.
- [ ] Test duplicate registration fails.
- [ ] Test unknown adapter returns clear error.

## Task 2: Use native-compatible targets first

- [ ] Add adapter metadata for MinIO S3: endpoint, access key env, secret key env, path-style setting.
- [ ] Add adapter metadata for Scylla Alternator DynamoDB: endpoint, dummy AWS credentials, region.
- [ ] Add adapter metadata for Redis: host, port, TLS, password.
- [ ] Add conformance smoke tests using SDK calls: put/get object, put/get item, set/get Redis key.

## Task 3: Build SQS-compatible HTTP adapter

- [ ] Implement AWS Query API actions: `CreateQueue`, `GetQueueUrl`, `SendMessage`, `ReceiveMessage`, `DeleteMessage`, `GetQueueAttributes`, `SetQueueAttributes`.
- [ ] Back it with RabbitMQ using existing queue naming.
- [ ] Preserve visibility timeout, delay seconds, retention period, DLQ mapping.
- [ ] Test with AWS SDK pointed at local endpoint.

## Task 4: Build Secrets Manager-compatible adapter

- [ ] Implement `GetSecretValue`, `DescribeSecret`, `ListSecrets`.
- [ ] Back it with Vault KV.
- [ ] Support secret names and ARNs generated by HomePort.
- [ ] Test with AWS SDK `secretsmanager.GetSecretValue`.

## Task 5: Build CloudWatch Logs ingest adapter

- [ ] Implement `CreateLogGroup`, `CreateLogStream`, `PutLogEvents`, `DescribeLogStreams`.
- [ ] Back it with Loki HTTP push.
- [ ] Preserve sequence token behavior loosely enough for SDK clients.
- [ ] Test with AWS SDK `cloudwatchlogs.PutLogEvents`.

## Task 6: Mark impossible/expensive compat honestly

- [ ] In the coverage ledger, mark Cognito full API compatibility as `Guided` until proven.
- [ ] Mark KMS key-material export as `Impossible`; support new Vault key plus re-encryption.
- [ ] Mark BigQuery/Spanner/Power BI/Bedrock-style products as `Missing` or `Guided` until adapters exist.
- [ ] Commit with `git commit -m "feat: add cloud API compatibility registry"`.
