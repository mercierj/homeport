# HomePort Storage Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make object, block, and file storage migrations click-bam across AWS, Google Cloud, and Azure.

**Architecture:** Object storage standardizes on S3-compatible MinIO. Block/file storage standardizes on volumes plus NFS/SMB where protocol compatibility is required. Data movement is runbook-driven with verification.

**Tech Stack:** Existing storage mappers, `internal/infrastructure/sync/minio.go`, rclone/mc where already used, Docker volumes, NFS/Samba containers.

---

## Scope

- AWS: S3, EBS, EFS
- Google Cloud: GCS, Persistent Disk, Filestore
- Azure: Blob Storage, Storage Account, Files, Managed Disk

## Task 1: Object storage no-rewrite path

- [ ] Update S3/GCS/Blob mappers to emit a shared `object-storage` runbook.
- [ ] Provision MinIO buckets, access keys, public policies, versioning, lifecycle metadata, and encryption settings.
- [ ] Generate SDK env vars for AWS SDK path-style endpoint.
- [ ] For GCS/Azure SDKs, either configure compatible endpoint if supported or mark API adapter required in coverage.
- [ ] Validate with put/list/get/delete object tests.

## Task 2: Object storage data migration

- [ ] Convert current scripts into runbook command steps.
- [ ] Add source object count and byte estimate.
- [ ] Sync source to MinIO.
- [ ] Verify object counts, sample checksums, and missing keys.
- [ ] Add rollback step that keeps source as authority until cutover passes.

## Task 3: Block storage

- [ ] For AWS EBS, GCP Persistent Disk, Azure Managed Disk, add snapshot discovery to runbook.
- [ ] Automate snapshot export/import where provider APIs allow it.
- [ ] If provider blocks direct export, create a guided step that launches a temporary helper VM and streams data.
- [ ] Validate filesystem mount and expected used bytes.

## Task 4: File storage

- [ ] For EFS/Filestore/Azure Files, provision NFS or SMB target based on source protocol.
- [ ] Automate rsync/robocopy-style copy.
- [ ] Validate file count, byte count, permissions sample, and mount from target app container.
- [ ] Replace mapper manual mount instructions with runbook mount validation.

## Task 5: UI

- [ ] Add object/block/file storage step cards to migration wizard.
- [ ] Show progress, transferred bytes, skipped files, failed files, and retry button.
- [ ] Do not show "manual instructions" unless the ledger marks the step `Impossible`.

## Task 6: Tests

- [ ] Run storage mapper tests.
- [ ] Add one integration fixture covering S3 + EBS + EFS equivalents.
- [ ] Commit with `git commit -m "feat: automate storage migration runbooks"`.
