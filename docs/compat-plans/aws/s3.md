# AWS S3 Local Compatibility Plan

## Goal

Provide an in-memory S3 surface for local endpoint-override checks without claiming MinIO deployment or durable object storage.

## Provider API Surface

- Supported operations cover bucket and object lifecycle, bucket tags, pagination, idempotency, quotas, authorization, and audit callbacks.
- State is held in memory and disappears when the adapter stops.
- Ledger resource types: `aws_s3_bucket`

## Backend

- Backend: MinIO.
- Local status: proposed migration seed only; MinIO is not deployed and persistence, HA, backup, validation, cutover, and rollback are not proved.

## Authz Model

- Every supported operation calls the shared authorizer with the bucket or object ARN.
- Denied requests do not mutate process-local state.

## Adapter

- Endpoint: `/compat/aws/s3`.
- Requests and responses use the S3 REST/XML shapes exercised by official clients.
- Unsupported operations return provider-shaped errors.

## Generated Artifacts

- `artifacts/compat/aws/s3/backend.yaml` records the proposed MinIO target.
- `artifacts/compat/aws/s3/adapter.yaml` records local actions, errors, pagination, idempotency, and quota behavior.
- `artifacts/compat/aws/s3/migration.md` preserves source identifiers without asserting migration execution.
- `test/conformance/services/aws-s3.yaml` records the runnable local contract.

## Contract Tests

- AWS SDK for Go v2 exercises bucket and object lifecycle, tags, pagination, idempotency, quota, authorization, and audit behavior against `/compat/aws/s3`.
- AWS CLI, Terraform, and Boto3 endpoint-override checks cover their documented local slices.

## Compatibility Level

- Current level: L3 local seed.
- Target level: L4 only after durable MinIO integration and external migration gates are proved.
- Blocking gaps: durable MinIO storage, production validation, cutover, and rollback.
